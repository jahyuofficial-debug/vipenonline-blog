# VIPEN ONLINE 网站重构计划

---

## 一、参考设计风格分析（Dribbble EternaCloud）

从设计稿分析，该设计具有以下特点：
- **深色主题**：黑色/深灰色背景，营造高端感
- **极简布局**：大留白，简洁的卡片式排版
- **现代字体**：无衬线字体，大标题
- **渐变元素**：微妙的渐变色彩点缀
- **动画效果**：平滑的过渡和交互动效
- **导航栏**：顶部简洁导航，带 Logo 和菜单项
- **Hero 区域**：全屏大标题 + 副标题 + CTA 按钮
- **内容分区**：清晰的区块划分

---

## 二、需求拆解

### 1. 网站视觉重设计
- 抛弃 Butterfly 默认样式，自定义 CSS 和 Pug 模板
- 实现参考 Dribbble 设计的深色极简风格
- 添加动画效果（滚动动画、悬停效果、渐变背景）

### 2. 素材替换指南
- 头像/Logo：`source/img/avatar.jpg`
- 封面图片：`source/img/covers/` 目录下
- 404 页面图片：`source/img/404.jpg`
- Favicon：`source/img/favicon.ico`
- 文章配图：在 Markdown 中引用 `/img/xxx.jpg`
- 视频存放：`source/videos/` 目录（需创建）

### 3. 暗号验证页面（核心功能）
- **功能**：用户访问网站前需要输入暗号
- **逻辑**：不同暗号 → 不同页面内容/权限
- **实现方式**：
  - 创建独立登录页面（覆盖层形式）
  - 暗号验证使用 JavaScript + localStorage（前端方案）
  - 或使用 Vercel Edge Functions / Cloudflare Workers（更安全）
  - 暗号存储在服务器端配置或环境变量中
- **页面路由**：
  - 无暗号/错误暗号 → 公共首页
  - 暗号 A → 展示特定页面组 A
  - 暗号 B → 展示特定页面组 B

### 4. 留言板系统
- **功能需求**：
  - 纯文字留言
  - 图文留言（支持图片上传）
  - 纯 Emoji 留言（内置 Emoji 选择器）
- **实现方案**：
  - **方案 A（推荐）**：集成 Twikoo 或 Waline 评论系统
    - 支持文字 + 图片 + Emoji
    - 免费开源，无需自建后端
    - 数据存储在 Vercel/KV 或 LeanCloud（免费）
  - **方案 B**：自研留言板
    - 需要后端 API（Vercel Serverless Functions）
    - 数据库（Vercel Postgres 或 Supabase 免费层）
    - 开发工作量大
- **Emoji 选择器**：使用 `emoji-mart` 或原生实现

## 七、关于 Dribbble 设计的实现

参考设计的核心元素可拆解为：
- **深色背景** + 微妙渐变光晕
- **大字体标题**（60px+）
- **圆角卡片**（border-radius: 16px+）
- **玻璃态效果**（backdrop-filter: blur）
- **微交互**（按钮 hover、卡片悬浮）

这些都可以用纯 CSS + Pug 模板实现，无需额外框架。

## 八、注意事项

1. **暗号验证的安全性**：前端验证可以被绕过，如需高安全性需配合后端验证
2. **图片加载优化**：大量图片需考虑 lazyload 和 WebP 格式
3. **SEO**：自定义模板需手动配置 meta 标签
4. **移动端适配**：响应式设计必须覆盖手机端
5. **留言板审核**：建议开启留言审核功能，防止垃圾内容
